📘 Setup Guide v1.0

AI Monitoring System
Setup Tutorial

Panduan lengkap untuk mengatur sistem monitoring penggunaan AI (Claude Code) — dari sisi Admin yang mengelola server sentral, hingga sisi Client yang menginstal agent di komputer masing-masing.

🧑‍💼
Admin (Kamu)
Mengelola server sentral, database PostgreSQL, admin dashboard, dan mendaftarkan device client.
💻
Client (Teman)
Menginstal local-agent di komputernya untuk memantau penggunaan Claude dan sinkronisasi ke server admin.
🗺️ Arsitektur Sistem
💻
Local Agent
:3002 — komputer client
HTTPS sync
setiap 30 detik
🖥️
Admin API
:4000 — server admin
query
🐘
PostgreSQL
:5433 — VPS
● Local Agent — baca log Claude di ~/.claude/projects/
● Admin API — terima sync, simpan ke PostgreSQL
● Admin Dashboard — visualisasi dari semua client
📦

Prasyarat

Software yang harus sudah terinstal sebelum memulai
Kedua pihak (Admin & Client) butuh:
  • ✓ Node.js v18+ — cek dengan node --version
  • ✓ pnpm — install dengan npm install -g pnpm
  • ✓ Git — untuk clone repository
  • ✓ Claude Code CLI — sudah terpasang dan pernah digunakan (menghasilkan log di ~/.claude/projects/)
ℹ️
Khusus Admin: Butuh akses ke VPS/server PostgreSQL. Tutorial ini mengasumsikan database sudah ada dan bisa diakses via connection string.

5. Daftarkan Client Baru

Langkah ini WAJIB dilakukan admin sebelum client bisa sync data
⚠️
Penting: Sistem ini tidak punya self-registration. Admin harus manual mendaftarkan setiap device client ke database, lalu memberikan Device ID dan API Key ke client.
1
Generate Device ID dan API Key untuk client
Buat dua UUID unik — satu untuk device_id, satu untuk api_key
bash 
# Generate di Linux/Mac:
$ cat /proc/sys/kernel/random/uuid  # → device_id
$ cat /proc/sys/kernel/random/uuid  # → api_key

# Atau pakai Node.js:
$ node -e "console.log(crypto.randomUUID())"  # jalankan 2x
2
Daftarkan device ke database PostgreSQL
Simpan script ini lalu jalankan dari root folder project. Ganti nilai sesuai data client.
bash — register-client.mjs 
// Simpan sebagai register-client.mjs lalu jalankan: node register-client.mjs

import postgres from './node_modules/.pnpm/postgres@3.4.9/node_modules/postgres/src/index.js';

const DB_URL  = 'postgresql://USER:PASSWORD@HOST:PORT/ai_monitoring';
const DEVICE  = {
  device_id  : 'UUID-DEVICE-ID-DARI-LANGKAH-1',   // ganti ini
  api_key    : 'UUID-API-KEY-DARI-LANGKAH-1',     // ganti ini
  name       : 'Laptop-Teman',                   // nama device
  hostname   : 'LAPTOP-TEMAN',                   // hostname komputer client
  os         : 'linux',                          // linux / win32 / darwin
  arch       : 'x64',
  version    : '1.0.0'
};

const sql = postgres(DB_URL);
const now = new Date().toISOString();

await sql`
  INSERT INTO devices (id, api_key_hash, name, hostname, os, arch, tracker_version, created_at, last_seen_at)
  VALUES (${DEVICE.device_id}, ${DEVICE.api_key}, ${DEVICE.name}, ${DEVICE.hostname},
          ${DEVICE.os}, ${DEVICE.arch}, ${DEVICE.version}, ${now}, ${now})
  ON CONFLICT (id) DO NOTHING
`;

await sql.end();
console.log('✅ Device berhasil didaftarkan!');
console.log('Kirim ke client:');
console.log('  device_id :', DEVICE.device_id);
console.log('  api_key   :', DEVICE.api_key);
bash — jalankan script 
$ node register-client.mjs
3
Kirim credential ke client
Setelah script berhasil, kirim 3 informasi ini ke teman kamu:
🔐 Credential untuk Client
device_id
UUID yang kamu generate di langkah 1
api_key
UUID yang kamu generate di langkah 1
sync_endpoint
http://<IP-SERVER-ADMIN>:4000
⚠️
Keamanan: Jangan share credential ini secara publik. Gunakan chat pribadi atau password manager. API key ini berfungsi seperti password.
4
Verifikasi device terdaftar
bash 
$ node --input-type=module <<'EOF'
import postgres from './node_modules/.pnpm/postgres@3.4.9/node_modules/postgres/src/index.js';
const sql = postgres('postgresql://USER:PASSWORD@HOST:PORT/ai_monitoring');
const rows = await sql`SELECT id, name, created_at FROM devices`;
console.log(rows);
await sql.end();
EOF

3. Setup Database Lokal (SQLite)

Database lokal otomatis dibuat saat agent pertama kali jalan
1
Database SQLite dibuat otomatis di ~/.ai-usage/local-tracker.db
Tidak perlu install PostgreSQL di komputer client. Jika ingin memastikan tabel sudah ada sebelum menjalankan agent, bisa jalankan perintah berikut (opsional):
bash — dari root folder project 
$ node --input-type=module <<'EOF'
import Database from './node_modules/.pnpm/better-sqlite3@11.10.0/node_modules/better-sqlite3/lib/index.js';
import { homedir } from 'os';
import { join } from 'path';
import { mkdirSync, existsSync } from 'fs';

const dir = join(homedir(), '.ai-usage');
if (!existsSync(dir)) mkdirSync(dir, { recursive: true });
const db = new Database(join(dir, 'local-tracker.db'));
db.pragma('journal_mode = WAL');
db.pragma('foreign_keys = ON');

db.exec(`
CREATE TABLE IF NOT EXISTS events (
  id TEXT PRIMARY KEY, device_id TEXT NOT NULL, tool TEXT NOT NULL,
  integration TEXT NOT NULL, integration_version TEXT, project_hash TEXT,
  project_alias TEXT, session_id TEXT, message_id TEXT, timestamp TEXT NOT NULL,
  started_at TEXT, ended_at TEXT, duration_ms INTEGER, provider TEXT, model TEXT,
  input_tokens INTEGER, output_tokens INTEGER, total_tokens INTEGER,
  token_count_status TEXT NOT NULL, pricing_rule_id TEXT, estimated_cost_usd REAL,
  event_type TEXT NOT NULL, sync_mode TEXT NOT NULL, metadata TEXT,
  created_at TEXT NOT NULL, updated_at TEXT NOT NULL
);
CREATE TABLE IF NOT EXISTS sync_queue (
  event_id TEXT PRIMARY KEY REFERENCES events(id), status TEXT NOT NULL,
  retry_count INTEGER NOT NULL DEFAULT 0, next_retry_at TEXT,
  last_attempt_at TEXT, last_error TEXT, created_at TEXT NOT NULL, updated_at TEXT NOT NULL
);
CREATE TABLE IF NOT EXISTS projects (hash TEXT PRIMARY KEY, alias TEXT, created_at TEXT NOT NULL, updated_at TEXT NOT NULL);
CREATE TABLE IF NOT EXISTS sessions (id TEXT PRIMARY KEY, project_hash TEXT, start_time TEXT NOT NULL, end_time TEXT, created_at TEXT NOT NULL, updated_at TEXT NOT NULL);
CREATE TABLE IF NOT EXISTS settings (key TEXT PRIMARY KEY, value TEXT);
CREATE TABLE IF NOT EXISTS devices (id TEXT PRIMARY KEY, name TEXT NOT NULL, hostname TEXT NOT NULL, os TEXT NOT NULL, arch TEXT NOT NULL, tracker_version TEXT NOT NULL, created_at TEXT NOT NULL, last_seen_at TEXT NOT NULL);
CREATE TABLE IF NOT EXISTS pricing_rules (id TEXT PRIMARY KEY, provider TEXT NOT NULL, model TEXT NOT NULL, input_price_per_1m_tokens REAL NOT NULL, output_price_per_1m_tokens REAL NOT NULL, currency TEXT NOT NULL DEFAULT 'USD', effective_from TEXT NOT NULL, effective_to TEXT, created_at TEXT NOT NULL, updated_at TEXT NOT NULL);
`);

db.close();
console.log('✅ Database lokal siap di ~/.ai-usage/local-tracker.db');
EOF

5. Verifikasi Sync Berhasil

Cara memastikan data sudah terkirim ke server admin
1
Cek log local-agent
Tunggu ~30 detik setelah agent pertama kali jalan, lalu perhatikan log:
output yang benar 
✅ BERHASIL:
[SyncEngine] Successfully synced 8 events to Admin API.

❌ MASIH GAGAL — cek credential:
Sync error: Error: Sync failed with status 403
→ Berarti api_key tidak cocok. Hubungi admin untuk cek ulang.

❌ SERVER TIDAK BISA DIJANGKAU:
Sync error: Error: fetch failed
→ Berarti sync_endpoint salah atau server admin tidak berjalan.
2
Cek via curl (opsional)
bash 
# Test koneksi ke admin-api:
$ curl http://IP-SERVER-ADMIN:4000/api/health
→ harus return: {"status":"ok","version":"1.0.0"}

# Test auth dengan api_key:
$ curl -X POST http://IP-SERVER-ADMIN:4000/api/sync/batch \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer API-KEY-KAMU" \
  -d '{"events":[]}'
→ harus return: {"accepted":0,"duplicates":0,"failed":[],...}
✓
Setup selesai! Admin bisa melihat data kamu di dashboard
🎉
Data penggunaan Claude kamu sekarang ter-sync ke admin dashboard setiap 30 detik. Admin bisa membuka http://localhost:3000 (atau URL server admin) untuk melihat statistik.
🔄

Alur Sinkronisasi Data

Bagaimana data mengalir dari Claude di komputermu ke admin dashboard
Kamu pakai Claude Code CLI
Claude otomatis menulis log session ke ~/.claude/projects/<project>/<session>.jsonl
Local Agent membaca log (setiap 5 detik)
Daemon membaca file .jsonl, mengekstrak token usage, model, timestamp, dan menyimpan ke SQLite lokal di ~/.ai-usage/local-tracker.db
Sync Engine mengirim batch ke admin (setiap 30 detik)
Mengambil max 100 events dari sync queue dengan status "pending", mengirim ke POST /api/sync/batch dengan Bearer token
Admin API memvalidasi dan menyimpan
Memeriksa API key, menyimpan events ke PostgreSQL, mencatat sync log. Mengembalikan {"accepted": N}
Admin Dashboard menampilkan statistik
Refresh otomatis setiap 5 detik. Menampilkan total token, biaya estimasi, grafik per-hari, dan breakdown per model dari semua client

Sync Mode — Kontrol Privasi

Kontrol seberapa banyak data yang dikirim ke admin
Mode Data Dikirim Kelebihan Kekurangan
metadata_only Token count, model, timestamp, project hash ✅ Privasi terjaga, payload kecil Admin tidak bisa lihat konteks percakapan
redacted_content Isi chat dengan data sensitif tersensor Balance antara privasi dan transparansi ⚠️ Fitur masih dalam pengembangan
full_content Semua percakapan lengkap dengan Claude Admin bisa audit penggunaan AI detail ⚠️ Privasi rendah — gunakan hanya jika ada policy khusus

Troubleshooting

Solusi untuk masalah umum
!
Error: Sync failed with status 403
API key tidak cocok atau device belum terdaftar di database admin.
  • →Cek isi api_key di ~/.ai-usage/config.json — harus sama persis dengan yang di-input saat registrasi
  • →Admin: cek tabel devices di PostgreSQL — pastikan api_key_hash terdaftar
  • →Restart local-agent setelah memperbaiki config
!
Error: ECONNREFUSED / fetch failed
Tidak bisa terhubung ke admin-api.
  • →Pastikan admin-api sedang berjalan di server
  • →Cek sync_endpoint di config — gunakan IP publik server, bukan localhost
  • →Cek firewall: port 4000 harus terbuka
!
Events tidak muncul di dashboard admin
  • →Tunggu minimal 30-60 detik untuk sync cycle pertama
  • →Cek log local-agent — harus ada baris [SyncEngine] Successfully synced N events
  • →Jika stuck di "sending" — restart local-agent (on-start auto-recover events stuck di status sending)
!
Dashboard menampilkan 0 data
  • →Pastikan Claude Code CLI pernah digunakan — log ada di ~/.claude/projects/
  • →Ganti time range di dashboard dari "Today" ke "All"
  • →Cek DATABASE_URL di dashboard mengarah ke database yang benar
!
pnpm install error: Invalid package manager specification
  • →Edit package.json di root: ubah "version": "^11.5.3" → "version": "11.5.3"
